## 2. Stockfish Engine UCI Integration

The Dart UI interacts with the Stockfish processes using standard UCI commands. The new and updated options are passed dynamically upon starting or resetting an engine instance:

1. **Transposition Table (Hash) Size**:
   ```uci
   setoption name Hash value <hashSize>
   ```
   *Enforces the memory limit mapped to each avatar (e.g., `16`, `32`, `128`, or `256` MB).*

2. **Contempt Options**:
   ```uci
   setoption name Contempt value <contempt>
   ```
   *Sends the positive or negative contempt values configured to shape aggressive draw-avoidance or defensive draw-seeking behaviors. Neutral personas pass a default value of `0`.*

## 3. In-Depth Persona Behavioral Heuristics (Rust Core)

Move evaluations generated by Stockfish are adjusted by custom Rust heuristics to enforce distinct play styles. The final move selected is determined by the highest score calculated as:
$$\text{Score} = \text{Stockfish Evaluation} + \text{Heuristic Weight}$$

The detailed weights, gate thresholds, and behavior rules implemented in [persona.rs](file:///c:/Users/Public/Documents/ICA/rust_core/src/api/persona.rs) are described below:

### Sparky (Absolute Beginner)
* **Blunder Gate**: `9999.0` (Accepts moves of any evaluation quality)
* **Jitter Range**: `[-0.80, 0.80]` pawns (`get_jitter_amount(hash, 16000, 80.0)`)
* **Behavior Rules**: Adds massive evaluation noise to all legal moves, causing completely random woodpushing.

### Pawzy *(Pawnzy)*
* **Blunder Gate**: `5.0` pawns
* **Jitter Range**: `[-0.30, 0.30]` pawns (`get_jitter_amount(hash, 6000, 30.0)`)
* **Behavior Rules**:
  * Pushing pawns gets `+2.0` weight.
  * Moving non-king, non-pawn pieces gets a `-1.5` weight penalty.

### Timorous *(Coward)*
* **Blunder Gate**: `3.0` pawns
* **Jitter Range**: `[-0.10, 0.10]` pawns (`get_jitter_amount(hash, 2000, 10.0)`)
* **Behavior Rules**:
  * Retreating pieces gets `+1.2` weight.
  * Castling gets `+1.5` weight.
  * Supported/defended destination squares get `+1.0` weight.
  * Pushing non-advanced pawns (rank < 5 for white, rank > 4 for black) gets `+0.8` weight.
  * Capturing opponent units gets a `-0.5` weight penalty.
  * Advancing pieces deep into opponent territory (rank >= 5 for white, rank <= 4 for black) gets a `-1.0` weight penalty.

### Rookie
* **Blunder Gate**: `2.0` pawns
* **Jitter Range**: `[-0.06, 0.06]` pawns (`get_jitter_amount(hash, 1200, 6.0)`)
* **Behavior Rules**:

## 4. Discovered String Mismatches (UI vs Rust Core)

> [!WARNING]
> There are active mismatches between the avatar names configured on the Dart UI side and what the Rust core expect. This causes some personas to play under default settings instead of their custom behaviors.
>
> 1. **Timorous vs Coward**: The Dart UI names the avatar `'Timorous'`, but the Rust engine matching rules in `persona.rs` and `assignment.rs` look for `"Coward"`. Because there is no `"Timorous"` case, it hits the fallback case. This defaults Timorous' settings to `multi_pv: 4, skill_level: 10, depth: 8` and a blunder gate threshold of `9999.0` (acting like a medium-strength chaotic bot).
> 2. **Pawzy vs Pawnzy**: The Dart UI uses `'Pawzy'`. While `persona.rs` maps `"Pawzy" | "Pawnzy"` correctly, `assignment.rs` only defines `"Pawnzy"`, causing mismatch behavior in assignments.
>
> **Recommended Fix**: Add `"Timorous"` as an explicit match alias wherever `"Coward"` is evaluated in `persona.rs` and `assignment.rs` so that it plays using its defensive retreats profile.
